# 完整旅行规划

用这个 skill 做旅行规划时，目标不是写泛泛攻略，而是交付一份能执行的路线：为什么这样走、每天怎么走、路上看什么、住哪里、大概要花多少钱、哪些信息需要复核。

v1 强能力边界：中国境内路线。中国境内用小红书、中文网络攻略、高德 MCP、酒店聚合和费用估算闭环。海外目的地也可以规划，但要用当地语言加英文搜索，并说明高德路线验证不可作为海外主依据。

## 输入要求

先提取：

- 出发地：城市、机场、车站或详细地址。
- 目的地或方向：终点、区域、景区、环线主题；如果只说“云南/川西”等大区域且无法判断核心目的地，追问。
- 出行方式：自驾、公共交通、火车/飞机接驳、包车、骑行等。缺失时追问。
- 可选：出行日期、可用天数、人数、预算、偏好、住宿要求、同行人、是否往返、是否避开高速/夜路/高反/拥堵。

如果没有指定天数，不要先追问天数；先根据攻略常见线路、总里程、交通方式和每日强度推断推荐天数，并说明依据。如果出发地、目的地/方向或出行方式缺失，先简短追问。

## Preflight

开始前做来源健康检查，只输出状态和原因，不回显完整 key 或 cookie：

- 小红书：检查 `/Users/whisper/Desktop/workplace/xiaohongshu-skills/SKILL.md`、`/Users/whisper/Desktop/workplace/xiaohongshu-skills/scripts/cli.py`、项目 `.venv`/`uv`/`python3`、Chrome/扩展/Bridge。小红书必须走浏览器自动化路径：`scripts/cli.py` 只是控制真实浏览器页面，不是绕开浏览器的接口。先进入小红书项目目录再运行命令：优先 `cd /Users/whisper/Desktop/workplace/xiaohongshu-skills && ./.venv/bin/python scripts/cli.py check-login`，没有虚拟环境时用 `cd /Users/whisper/Desktop/workplace/xiaohongshu-skills && uv run python scripts/cli.py check-login`，最后才用 `cd /Users/whisper/Desktop/workplace/xiaohongshu-skills && python3 scripts/cli.py check-login`。不要在当前旅行 skill 仓库误跑相对 `scripts/cli.py`。未登录、扫码验证或 bridge 不可用时，打开/复用可见浏览器让用户处理，再继续。
- 图片读取：确认当前 agent 可以下载图片到本地并用视觉能力读取；若当前环境无法直接读图，使用 `codex-vision-delegation` 或让用户提供可读图片路径。不要忽略小红书图片。
- 高德 MCP：确认 `maps_geo`、`maps_direction_driving`、`maps_distance`、`maps_around_search` 等可用。
- 酒店：确认 `hotel-search-cn` 存在；多日住宿会由它完成来源健康检查和比价。
- 辅助旅行来源：FlyAI、同程程心、携程问道、普通网页搜索是否可用。
- 本地命令：`python3`、`node`、`curl`、`jq`、`rg`。

如果小红书、图片读取或高德这三个核心能力有两个以上不可用，先修复或请用户处理；除非用户明确降级继续，不要直接输出最终行程。

## Workflow

1. 明确出发地、目的地/方向、交通方式和关键约束。
2. 执行 Preflight，必要时先处理登录、bridge、MCP 或 key 问题。
3. 并行搜集攻略：小红书为高权重来源，但小红书浏览器详情采集要限速且单 agent 串行；普通网页和当地语言搜索可并行补充；读取小红书图片和重要网页图片。
4. 从文本和图片中抽取候选路线、沿途点、避坑、季节路况、费用、住宿区域和时间建议。
5. 用高德 MCP 验证起终点、途经点、每日驾驶/步行/公交可达性、景区周边 POI、住宿锚点和绕路成本。
6. 生成 1-3 条大体路线，按风景密度、沿途有趣程度、驾驶强度、住宿可行性、预算和用户偏好评分，选主方案并保留备选。
7. 将主路线拆成每日计划；每一天包含出发/到达、途经点、预计距离/时间、游玩重点、餐饮/补给/加油/停车提醒。
8. 路线稳定后，为每天住宿点调用 `hotel-search-cn`，传入日期、人数、交通方式、住宿锚点和偏好。
9. 汇总费用表：酒店、油费、过路费、门票、停车/接驳、餐饮、备用费用；区分实价、攻略价和估算。
10. 输出报告，不隐藏来源失败、路线不确定性、季节/天气/路况风险和价格需复核项。

详细来源调用见 `references/source-playbooks.md`。路线和费用规则见 `references/route-and-costing.md`。并行/子 agent 调度见 `references/agent-orchestration.md`。

## Guardrails

- 不要只凭单篇攻略定路线；至少尝试小红书 + 一个补充来源 + 高德验证。
- 不要把图片当装饰；图片里的路线截图、景区地图、菜单价目表、停车信息和路况提示都要读取并转成事实。
- 不要把小红书当普通 HTTP/API 抓取源；它是浏览器自动化来源。不要让多个 agent 同时操作小红书详情页或同一个可见浏览器。
- 不要用直线距离替代自驾/公共交通时间；能用路线规划时必须用路线规划。
- 不要让酒店子 agent 决定最终总路线；酒店结果用于确认住宿可行性和预算。
